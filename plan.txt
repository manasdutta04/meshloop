MANAS — DAY 1: Setup + GitHub Models + Ingestion (Multi-file & ZIP support)
What must be done today
By end of Day 1: a folder or ZIP file containing multiple mixed CSV/Excel/PDF/JSON files can be uploaded and parsed into a structured memory dict. Both API connections work.
________________

M1.1 — Create project structure (30 min)
Create this exact folder layout on your machine:
meshloop/
├── agents/
│   ├── __init__.py
│   ├── ingestion.py      ← Parses ZIP/multiple files into memory dicts
│   ├── cleaning.py       ← Data cleaning & type normalization
│   ├── discovery.py      ← Statistical checks + LLM-based root causes
│   ├── chat.py           ← Multi-turn Incident Q&A using vector context
│   └── reporter.py       ← Forensic reports + chart configurations
├── utils/
│   ├── __init__.py
│   ├── llm.py            ← GitHub Models connection + local cache
│   └── vector_store.py   ← In-memory ChromaDB vector store
├── sample_data/          ← Test datasets (messy sales CSV + server reports PDF)
├── app.py                ← Streamlit UI (Priya's frontend)
├── pipeline.py           ← Master orchestrator (chains agents)
├── requirements.txt
├── .env
└── .gitignore

M1.2 — Get GitHub Models API access & Test (20 min)
Follow instructions above. Test GITHUB_TOKEN connectivity by running:
python test_connection.py

M1.3 — Build the LLM utility (1.5 hours)
utils/llm.py — wraps GITHUB_TOKEN connections with rate-limit retries and local disk caching to avoid repeating API calls.

M1.4 — Build Ingestion Agent with Multi-File support (2.5 hours)
agents/ingestion.py — parses a list of files or a ZIP archive containing mixed formats.
Stores:
* `dataframes`: Dictionary of filename -> pd.DataFrame (for CSV/Excel/JSON)
* `corpora`: Dictionary of filename -> text (for PDF/TXT)
* `metadata`: Metadata parameters (file names, sizes, counts)

Test it on a ZIP file containing:
1. `sales.csv`
2. `server_log.pdf`

MANAS — DAY 2: Cleaning Agent & Autonomous Root-Cause Discovery Agent
What must be done today
By end of Day 2: Tabular data is cleaned, and the Discovery Agent autonomously runs statistical checks to detect anomalies and queries ChromaDB to find contextual text explanations.
________________

M2.1 — Set up Semantic Kernel with GitHub Models (1.5 hours)
utils/sk_kernel.py — configures Semantic Kernel connectors pointing to the GitHub Models endpoint.

M2.2 — Build Cleaning Agent (2.5 hours)
agents/cleaning.py — standardizes and cleans dataframes (dates, currencies, nulls) and unstructured texts (extra carriage returns, whitespaces).

M2.3 — Build Discovery Agent with Root-Cause Linking (3 hours)
agents/discovery.py:
1. Statistical Anomaly Detector: Detects specific dates, regions, or columns with metrics falling outside normal ranges (IQR outliers or sudden >30% drops).
2. Semantic Search Context: For each anomaly, queries ChromaDB for document logs mentioning the anomaly date or region.
3. Explainer Agent: Feeds the anomaly metadata and the matching document context to GPT-4o to synthesize the Root Cause.

Format returned by `discover_patterns()`:
```json
{
  "insights": [
    {
      "type": "root_cause_anomaly",
      "severity": "critical",
      "title": "East region conversion drop due to database timeout",
      "description": "Sales in East region dropped 48% on May 12. Cross-referencing matched this with server_outages.pdf documenting a database connection timeout.",
      "data_evidence": {"date": "2026-05-12", "region": "East", "cost": 12400}
    }
  ],
  "top_insight": {...},
  "summary": "...",
  "total_found": 1
}
```

MANAS — DAY 3: Metadata Vector Store + Cross-Modal Chat Agent + Reporter
What must be done today
By end of Day 3: Chunks and insights are stored in ChromaDB with metadata tags (date, region, type). The chat agent answers queries by joining structured statistics and unstructured documents.
________________

M3.1 — Vector Store with Metadata Tagging (1.5 hours)
utils/vector_store.py — Stores chunks of the datasets.
* Tabular summaries and rows are tagged with `{"type": "row", "date": "...", "region": "..."}`
* Text logs paragraphs are tagged with `{"type": "log", "date": "...", "region": "..."}`

M3.2 — Cross-Modal Chat Agent (1.5 hours)
agents/chat.py — Performs multi-turn Q&A. Takes a query, retrieves contextual chunks from ChromaDB, and prompts GPT-4o to answer citing specific documents (e.g. `[server_outages.pdf, page 3]`).

M3.3 — Reporter Agent (1 hour)
agents/reporter.py — Generates a markdown "Forensic Audit Report" summarizing anomalies and explaining the matching root causes.

M3.4 — Master Pipeline (45 min)
pipeline.py — coordinates the full 5-step ARCA flow.
